# Operations Guide

## Install for development

```bash
cd vaeru-ai
python3 -m venv .venv
. .venv/bin/activate
pip install -e .
vaeru init
```

## Run one cycle

```bash
vaeru tick
```

Run a safe synthetic demo:

```bash
vaeru demo
vaeru threats
vaeru actions
```

## Run as a foreground daemon

```bash
vaeru daemon --interval 2
```

Stop with `Ctrl-C`.

## Inspect state

```bash
vaeru status
vaeru residues --limit 50
vaeru threats --limit 50
vaeru actions --limit 50
vaeru topology
```

For machine-readable output, put `--json` before the subcommand:

```bash
vaeru --json status
vaeru --json threats --limit 5
```

## Configuration

Create `vaeru.toml` with:

```bash
vaeru init
```

Important sections:

- `[runtime]`: home directory, tick interval, cooldowns.
- `[policy]`: safety gates and action permissions.
- `[decay]`: residue half-life and accumulation gain.
- `[echo]`: echo encoding and resonance thresholds.
- `[surfaces]`: enabled collectors and filesystem watch paths.

## Evidence files

Detection notes are JSON files under:

```text
.vaeru/evidence/
```

Each note contains:

- Threat object fields.
- Causal chain and confidence.
- Defensive complements generated by P6.
- Action rationale.
- Operator safety note.

## Optional systemd helper

`scripts/install-systemd.sh` installs a conservative service unit for a dedicated runtime home. Review the script and generated unit before use.

Example:

```bash
sudo scripts/install-systemd.sh /opt/vaeru /var/lib/vaeru
sudo systemctl start vaeru
sudo systemctl status vaeru
```

The service runs the Python package already installed in the selected virtual environment. It does not grant packet, firewall, kill, or ptrace capabilities.

## Troubleshooting

### No threats emitted

That is normal on a quiet host. Try:

```bash
vaeru demo
```

### Too many residues

Increase decay or raise thresholds in `vaeru.toml`:

```toml
[decay]
residue_half_life_seconds = 120

[policy]
threat_escalation_weight = 0.75
```

### Evidence contains unexpected process command lines

Reduce process collection by disabling proc surface collection:

```toml
[surfaces]
enable_proc = false
```

Or run VAERU only on lab/staging hosts until collection scope is approved.
